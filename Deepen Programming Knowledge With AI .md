

# Applying AI to Deepen Programming Language Understanding

---

## Activity 1: Idiomatic Code Transformation

### Task

Modify code to make it more idiomatic in the selected language.

### Process

* Selected a recent function (15–30 lines)
* Used AI prompt to improve idiomatic style
* Compared original and improved version
* Applied suggested changes

### 3 Key Learnings

* Idiomatic code is easier to read and understand.
* Language features (built-in functions, short syntax) reduce complexity.
* Best practices help avoid rewriting features that already exist.

---

## Activity 2: Code Quality Review

### Task

Review older code and improve it.

### Process

* Selected older code (3+ months old)
* Used AI prompt to find issues
* Created a checklist from results
* Used checklist for future improvements

### 3 Key Learnings

* Code smells (long methods, repeated logic) make code harder to maintain.
* Readability helps find and fix bugs faster.
* Structured feedback improves performance and maintainability.

---

## Activity 3: Understanding Language Feature

### Task

Learn an advanced or unfamiliar language feature.

### Process

* Selected a new feature (e.g. decorators, generators, etc.)
* Used AI to explain with examples
* Learned use cases and mistakes
* Built a small example
* Got AI feedback

### 3 Key Learnings

* Advanced features make complex logic simpler when used correctly.
* Practical examples help understand when to use a feature.
* Mistakes happen when advanced features are used unnecessarily.

---

## Group Discussion (Summary Notes)

### Common Themes

* Readability is more important than clever code.
* AI helps find improvements and best practices.
* Refactoring improves maintainability more than writing new code.
